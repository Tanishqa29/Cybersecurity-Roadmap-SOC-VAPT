# Episode 01 - Enterprise Network Segmentation

## Overview

Network segmentation is one of the most important security controls in an enterprise environment. Instead of allowing every device to communicate freely, the network is divided into smaller, controlled segments. This limits the movement of attackers and reduces the impact of security incidents.

---

## Why It Matters

Without segmentation:

- Malware spreads quickly
- Attackers move laterally across the network
- Sensitive systems become easier to compromise

With segmentation:

- Better security
- Reduced attack surface
- Easier monitoring
- Improved compliance

---

## Enterprise Scenario

Imagine a company with:

- HR Department
- Finance Department
- IT Department
- Guest Wi-Fi
- Data Center

Each department should operate in its own isolated network.

---

## Common Segments

- User Network
- Server Network
- Management Network
- DMZ
- Guest Network
- Production Environment

---

## Security Controls

- VLANs
- Firewalls
- Access Control Lists
- Zero Trust Policies
- Network Monitoring

---

## Best Practices

- Never allow unrestricted communication.
- Separate critical assets.
- Apply least privilege.
- Monitor east-west traffic.
- Review firewall rules regularly.

---

## Key Takeaways

Network segmentation reduces the attack surface and prevents attackers from moving freely across enterprise environments.
